**Goal:** Дать пользователю выбор режима CI: `wide` (5/95 перцентили, вся история) или `narrow` (25/75 перцентили, последние 3 года).

**Architecture:** Вся логика выбора инкапсулирована в helper `_get_ci_params(ci_mode, X_train, y_train)` — возвращает `(X_q, y_q, lower_alpha, upper_alpha)`. `prepare_and_train_model` получает новый параметр `ci_mode: str = 'wide'`. UX: новый вопрос `5.` в `get_user_inputs()` и флаг `--ci-mode` в argparse.

**Tech Stack:** Python, XGBoost quantile regression, argparse

```python
def _get_ci_params(ci_mode: str, X_train: np.ndarray, y_train: np.ndarray):
    """
    Возвращает (X_q, y_q, lower_alpha, upper_alpha) для обучения квантильных моделей.
    narrow: последние 756 строк (~3 торговых года), перцентили 25/75.
    wide:   полная история, перцентили 5/95.
    """
    NARROW_WINDOW = 756
    if ci_mode == 'narrow':
        q_start = max(0, len(X_train) - NARROW_WINDOW)
        return X_train[q_start:], y_train[q_start:], 0.25, 0.75
    return X_train, y_train, 0.05, 0.95
```

```markdown
## [v14.1] — 2026-04-28

### Добавлено

- Выбор режима доверительных интервалов (`ci_mode`):
  - **Широкий** (`wide`, по умолчанию) — 5/95 перцентили, полная история обучения
  - **Узкий** (`narrow`) — 25/75 перцентили, последние 756 торговых дней (~3 года)
- Интерактивный вопрос `5.` в `get_user_inputs()`
- CLI-флаг `--ci-mode wide|narrow`
- Helper `_get_ci_params(ci_mode, X_train, y_train)` инкапсулирует логику выбора

### Причина

Широкий CI (5/95 на полной истории) учитывал кризис SBER 2022 и давал нижнюю границу
~260 при цене ~320 — статистически верно, но практически неинформативно.
Узкий режим использует актуальную волатильность последних 3 лет.
```
