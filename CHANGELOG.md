## [v14.1] — 2026-04-28

### Добавлено

- Выбор режима доверительных интервалов (`ci_mode`):
  - **Широкий** (`wide`, по умолчанию) — 5/95 перцентили, полная история обучения
  - **Узкий** (`narrow`) — 25/75 перцентили, последние 756 торговых дней (~3 года)
- Интерактивный вопрос `5.` в `get_user_inputs()`
- CLI-флаг `--ci-mode wide|narrow`
- Helper `_get_ci_params(ci_mode, X_train, y_train)` инкапсулирует логику выбора

### Исправлено

- Прогноз начинался со следующего дня после даты запуска, пропуская текущий торговый день.
  Причина: `base_date = end_date` (сегодня), но MOEX возвращает данные только до вчера.
  Теперь `base_date = data['Date'].max()` — последняя дата в фактических данных.
  Прогноз day 1 теперь = следующий рабочий день после последней известной цены (без пропуска).

### Причина

Широкий CI (5/95 на полной истории) учитывал кризис SBER 2022 и давал нижнюю границу
~260 при цене ~320 — статистически верно, но практически неинформативно.
Узкий режим использует актуальную волатильность последних 3 лет.

## [v14.0] — 2026-04-27

### Изменено (архитектурно)

- `stock_modelv13.py` переименован в `stock_modelv14.py` — архитектурный мейлстоун
- `prepare_and_train_model` — авторегрессивный прогноз на 2–3 дня заменён прямыми независимыми моделями на каждый горизонт (Direct Multi-Step Forecasting / MIMO):
  - Добавлен параметр `horizon: int = 1`
  - При подготовке обучающих последовательностей цель сдвигается: `y[i] = Close[i + horizon]`; верхняя граница цикла `len(scaled_df) - horizon`
  - Прогнозный блок: одно прямое предсказание из текущих данных вместо авторегрессивного цикла; синтетические строки Open/High/Low/Volume удалены
  - В `main` и `run_backtest` функция вызывается трижды (horizon=1,2,3); результаты мержатся в словари `forecasts` и `confidence_intervals`
  - Итого моделей: 54 (18 на горизонт × 3) вместо 18

### Причина

Авторегрессивный подход накапливал ошибку: для прогноза на день 2 и 3 модель получала синтетические Open/High/Low/Volume, а технические индикаторы (RSI, ATR, Bollinger Bands и др.) пересчитывались на основе этих фиктивных значений. Ошибка дня 1 входила в признаки дня 2, ошибка дня 2 — в признаки дня 3. Доверительные интервалы для горизонтов 2 и 3 были статистически несостоятельны.

### Плюсы нового подхода

- Нет накопления ошибки — каждый горизонт предсказывается независимо из реальных данных
- Честные CI — квантильные модели каждого горизонта обучены на соответствующем сдвиге цели
- Академически обоснован: Direct Multi-Step / MIMO forecasting — стандарт для горизонтов > 1
- Интерпретируемые метрики: RMSE/MAE/R² отдельные для каждого горизонта (показывают деградацию качества с ростом горизонта)
- Внешний интерфейс (`forecasts[h][-1]`, `confidence_intervals[h]`) не изменился — код вывода не тронут

### Минусы / компромиссы

- Время обучения ×3 (54 модели вместо 18)
- Три независимых ансамбля без разделяемого представления между горизонтами

## [v13.6] — 2026-04-22 02:30

### Добавлено

- `stock_modelv13.py` — 5 новых признаков на основе доходностей (returns):
  - `Vol_Return_5`, `Vol_Return_10`, `Vol_Return_20` — скользящее стандартное отклонение дневных доходностей за 5/10/20 дней (волатильность режима)
  - `Return_MA_5`, `Return_MA_10` — скользящее среднее дневных доходностей за 5/10 дней (направление momentum на returns, не на ценах)
- Вектор признаков расширен с 29 до 34; оба определения `features` (основное и Optuna) обновлены
- `update_technical_indicators_single_row` обновлена: поддерживает вычисление новых признаков для авторегрессивного прогноза на 2–3 дня

### Причина

`SMA_10`/`EMA_20` работают на уровнях цен и коррелированы с масштабом. Признаки на returns захватывают:

- **Кластеризацию волатильности** (GARCH-эффект): высоковолатильные периоды сгруппированы — сигнал для модели о режиме рынка
- **Momentum на returns**: средняя доходность за период отражает тренд независимо от ценового уровня

Изменение следует из анализа ACF/PACF в отчёте по производственной практике.

## [v13.0] — 2026-04-21 23:48

### Добавлено

- Стэкинг-ансамбль LSTM + XGBoost + Meta-Learner
- Квантильная регрессия для доверительных интервалов (α = 0.05, 0.50, 0.95)
- Walk-forward валидация (3 сплита с расширяющимся окном)
- Оптимизация гиперпараметров через Optuna с персистентностью результатов
- Режим бэктеста: обучение до заданной даты, сравнение с реальными ценами
- Загрузка данных MOEX через `moexalgo` + fallback на прямой ISS REST API
- Обогащение фундаментальными данными через T-Bank Invest API
- 39 признаков: OHLCV + 17 технических индикаторов + 17 фундаментальных
- Интерактивный режим ввода параметров и CLI-режим через `argparse`
