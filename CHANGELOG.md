## [v13.3] — 2026-04-22

### Исправлено

- `stock_modelv13.py` — даты прогноза теперь сдвигаются на рабочие дни (`next_business_day`): прогноз на пятницу больше не указывает субботу как торговый день

### Изменено

- `stock_modelv13.py` — нулевые фундаментальные признаки закомментированы в обоих определениях `features`; активные (из Tinkoff API) сгруппированы отдельно

### Причина

Даты прогноза некорректно указывали выходные дни. Нулевые признаки убраны из вектора фич, чтобы не передавать модели шумовые константы. Комментарии с TODO указывают на `financemarker.ru` как источник для заполнения отключённых признаков.

## [v13.2] — 2026-04-22

### Исправлено

- `stock_modelv13.py` — ошибка отступа: блок `if backtest_mode / else` перенесён внутрь `if __name__ == '__main__':`; ранее код выполнялся на уровне модуля, что ломало импорт
- `stock_modelv13.py` — Optuna для XGBoost теперь оценивает качество на holdout-выборке (20% от train), а не на тех же данных, на которых обучает; устранено in-sample переобучение при подборе гиперпараметров

### Добавлено

- `stock_modelv13.py` — функция `get_usd_rub_rate()`: актуальный курс USD/RUB загружается с API ЦБ РФ вместо захардкоженного значения 82.91; при недоступности API используется fallback 90.0

### Причина

Исправление структурного бага и некорректной оценки в Optuna напрямую влияют на воспроизводимость и качество результатов модели. Динамический курс устраняет дрейф признака `value_usd` при изменении курса рубля.

## [v13.1] — 2026-04-22

### Добавлено

- `config.py` — все гиперпараметры LSTM и XGBoost вынесены из `.env` в Python-конфиг; безопасно коммитить
- `.gitignore` — исключены `.env`, директория `outputs/`, артефакты Python и IDE
- `CHANGELOG.md` — этот файл
- `architecture.md` — описание архитектуры проекта, потока данных, переменных окружения и порядка запуска
- `improvements.md` — анализ плюсов/минусов текущего решения и рекомендации

### Изменено

- `stock_modelv13.py` — гиперпараметры теперь импортируются из `config.py` вместо чтения через `os.getenv`
- `.env` — оставлены только API-токены (секреты); параметры моделей перемещены в `config.py`
- `README.md` — создан по образцу проектной документации

### Причина

Разделение секретов (токены) и конфигурации (гиперпараметры): токены не должны попадать в VCS, гиперпараметры — наоборот, полезно версионировать для воспроизводимости экспериментов.

## [v13.0] — начальная версия

### Добавлено

- Стэкинг-ансамбль LSTM + XGBoost + Meta-Learner
- Квантильная регрессия для доверительных интервалов (α = 0.05, 0.50, 0.95)
- Walk-forward валидация (3 сплита с расширяющимся окном)
- Оптимизация гиперпараметров через Optuna с персистентностью результатов
- Режим бэктеста: обучение до заданной даты, сравнение с реальными ценами
- Загрузка данных MOEX через `moexalgo` + fallback на прямой ISS REST API
- Обогащение фундаментальными данными через T-Bank Invest API
- 39 признаков: OHLCV + 17 технических индикаторов + 17 фундаментальных
- Интерактивный режим ввода параметров и CLI-режим через `argparse`
