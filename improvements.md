## Плюсы текущего решения

### Архитектурные

- **Стэкинг-ансамбль LSTM + XGBoost** — LSTM улавливает временные зависимости и нелинейные паттерны в последовательностях; XGBoost хорошо работает с плоскими признаками и устойчив к выбросам. Meta-Learner объединяет их оптимально.
- **Квантильная регрессия для CI** — три квантильные модели (α=0.05, 0.50, 0.95) дают статистически обоснованные доверительные интервалы, а не просто ±N%.
- **Walk-forward валидация** — правильный способ кросс-валидации для временных рядов. Предотвращает утечку будущих данных (data leakage), в отличие от случайного разбиения.
- **Bayesian hyperparameter search (Optuna)** — эффективнее grid/random search. Персистентность результатов в JSON позволяет не перезапускать оптимизацию при повторных прогонах.
- **Режим бэктеста** — возможность честно проверить качество модели на прошлых данных, сравнивая прогноз с известным результатом.
- **Два источника данных** — `moexalgo` как основной, прямой MOEX ISS API как fallback. Повышает отказоустойчивость загрузки данных.
- **Обогащение фундаментальными данными** — интеграция с Tinkoff Invest API добавляет P/E, P/B, ROE и другие коэффициенты.
- **Конфигурация через .env** — гиперпараметры вынесены из кода, легко менять без правки скрипта.
- **Два режима запуска** — интерактивный (с подсказками) и CLI (для автоматизации и скриптов).
- **Сохранение всех артефактов** — логи, графики, гиперпараметры, результаты бэктестов структурированы по директориям.

## Минусы и проблемы

### Критические

**1. ✅ Ошибка отступа в точке входа (исправлено в v13.2)**

`if backtest_mode:` / `else:` (основной запуск модели) находились на уровне модуля, вне `if __name__ == '__main__':`. Перенесено внутрь блока в v13.2.

**2. ✅ SSL-верификация отключена (исправлено в v13.5)**

Оба вызова Tinkoff Invest API переведены на `verify=certifi.where()`. Добавлен `import certifi`. SSL-цепочка теперь проверяется по CA-пакету Mozilla.

**3. ✅ XGBoost Optuna оптимизировал на train-данных (исправлено в v13.2)**

Добавлен holdout-split 80/20 внутри функции оптимизации. Теперь Optuna оценивает XGBoost на 20% данных, которые не видел при обучении.

### Качество модели

**4. ✅ Большинство фундаментальных признаков = 0 (исправлено в v13.3)**

Нулевые фундаментальные признаки закомментированы в векторе `features`. Вектор сокращён с 39 до 29 признаков. TODO-комментарий указывает на `financemarker.ru` как источник данных.

**5. ✅ Курс USD/RUB захардкожен (исправлено в v13.2)**

Заменён функцией `get_usd_rub_rate()`, которая загружает актуальный курс с XML API ЦБ РФ (`cbr.ru`). При недоступности API используется fallback 90.0.

**6. ✅ Наивная авторегрессия при прогнозе на 2–3 дня (исправлено в v14.0)**

При прогнозе на день 2 новая строка создаётся как:

```python
'Open': prev_close,
'High': max(prev_close, pred_close) + volatility * 0.3,
'Low':  min(prev_close, pred_close) - volatility * 0.3,
'Volume': temp_df['Volume'].iloc[-1]  # просто копируется
```

Open/High/Low синтетически генерируются по простой формуле, Volume копируется без изменений — это нереалистично и снижает качество 2-х и 3-х дневных прогнозов.

**7. ✅ Скользящая волатильность и momentum доходностей (реализовано в v13.6)**

Добавлено 5 признаков на основе returns: `Vol_Return_5/10/20` (стд доходностей), `Return_MA_5/10` (среднее доходностей). Вектор расширен с 29 до 34 признаков. `update_technical_indicators_single_row` поддерживает новые признаки для авторегрессивного прогноза.

**8. Дубликат логики в `update_technical_indicators_single_row`**

Функция для обновления индикаторов одной строки полностью дублирует логику батчевого `update_technical_indicators`. При изменении любого индикатора нужно менять оба места — источник ошибок.

**9. ✅ Прогнозные даты не учитывали торговый календарь (исправлено в v13.3)**

Добавлена функция `next_business_day()`. Прогнозные даты теперь пропускают субботу и воскресенье.

### Производительность и ресурсы

**10. Нет управления памятью GPU**

TensorFlow по умолчанию занимает всю видеопамять. Без `tf.config.experimental.set_memory_growth(gpu, True)` при наличии GPU возможны OOM-ошибки при параллельном использовании ресурсов.

**11. Медленная валидация: 3 независимых обучения**

При walk-forward обучаются 3 полных ансамбля (LSTM + 4 XGBoost + meta-learner = 6 моделей × 3 сплита = 18 моделей). При использовании Optuna сверху ещё N_TRIALS обучений. На CPU с LSTM это очень медленно.

### Код и качество

**12. ✅ Мёртвый код и неиспользуемые импорты (исправлено в v13.4)**

Удалены неиспользуемые импорты: `TensorBoard`, `ModelCheckpoint`, `interp1d`, `io`, `Path`.

**13. Мета-learner не оптимизируется через Optuna**

Гиперпараметры мета-леарнера захардкожены:

```python
meta_params = {'n_estimators': 100, 'max_depth': 6, 'learning_rate': 0.03, ...}
```

При этом базовые модели оптимизируются Optuna. Непоследовательно.

**14. Дата начала данных жёстко задана**

```python
start_date = '2014-01-01'
```

Не настраивается через `.env` или CLI. Для тикеров с короткой историей (IPO после 2014) загружается пустой период в начале; для коротких экспериментов всегда грузятся 10+ лет данных.

**15. Логирование raw API response**

```python
logger.info(f"RAW API RESPONSE: {resp.text[:500]}...")
```

Первые 500 символов ответа API пишутся в лог-файл. Ответ может содержать токены или чувствительные данные.

## Что можно добавить / улучшить

### Высокий приоритет (влияют на качество)

1. ✅ **Правильная оценка XGBoost в Optuna** — реализовано в v13.2: holdout 80/20.

2. ✅ **Убрать нулевые фундаментальные признаки** — реализовано в v13.3: закомментированы. TODO указывает на `financemarker.ru` для дальнейшего заполнения.

3. ✅ **Динамический курс USD/RUB** — реализовано в v13.2: `get_usd_rub_rate()` с XML API ЦБ РФ.

4. ✅ **Торговый календарь** — реализовано в v13.3: `next_business_day()` пропускает выходные.

### Средний приоритет (архитектура)

1. **Прямое многошаговое прогнозирование** — вместо авторегрессии обучить отдельные модели для каждого горизонта (1d, 2d, 3d). Это избегает накопления ошибки и устраняет необходимость в синтетическом Open/High/Low/Volume.

2. **Рефакторинг дублированной логики индикаторов** — объединить `update_technical_indicators` и `update_technical_indicators_single_row` в одну функцию, принимающую параметр `mode`.

3. ✅ **Исправить отступ `if backtest_mode:`** — реализовано в v13.2.

4. ✅ **Фиксация SSL** — реализовано в v13.5: `verify=certifi.where()` в обоих вызовах.

### Низкий приоритет (удобство)

1. ✅ **`requirements.txt`** — реализовано в v13.5: зафиксированы 11 прямых зависимостей с точными версиями.

2. **GPU memory growth** — добавить в инициализацию:

   ```python
   for gpu in tf.config.list_physical_devices('GPU'):
       tf.config.experimental.set_memory_growth(gpu, True)
   ```

3. **Configurable start date** — добавить переменную `DATA_START_DATE` в `.env` и аргумент `--start-date` в CLI.

4. **Оптимизация мета-леарнера** — включить его гиперпараметры в Optuna или хотя бы вынести в константы/env.

5. **Расширение горизонта прогноза** — сделать горизонт настраиваемым через `.env` или CLI (сейчас жёстко задано [1, 2, 3]).

6. **Прогресс-бар для Optuna** — вернуть `show_progress_bar=True` или использовать tqdm, чтобы пользователь видел прогресс долгой оптимизации.
