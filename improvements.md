# Анализ проекта: плюсы, минусы, рекомендации

## Плюсы текущего решения

- **Стэкинг-ансамбль LSTM + XGBoost** — LSTM улавливает временные зависимости; XGBoost устойчив к выбросам. Meta-Learner объединяет их оптимально.
- **Квантильная регрессия для CI** — три квантильные модели (α=0.05, 0.50, 0.95) дают статистически обоснованные доверительные интервалы на основе OOS-остатков.
- **Walk-forward валидация** — правильный способ кросс-валидации для временных рядов, предотвращает data leakage.
- **MIMO (Direct Multi-Step Forecasting)** — отдельная модель на каждый горизонт, нет накопления ошибки авторегрессии.
- **Bayesian hyperparameter search (Optuna)** — персистентность результатов в JSON, не нужно перезапускать оптимизацию.
- **Два источника данных** — `moexalgo` основной, прямой MOEX ISS API как fallback.
- **Обогащение фундаментальными данными** — T-Bank Invest API: P/E, P/B, ROE и другие коэффициенты.
- **Два режима запуска** — интерактивный и CLI (для автоматизации).
- **Сохранение всех артефактов** — логи, графики, гиперпараметры, результаты бэктестов.

## Открытые задачи

### Производительность

- **GPU memory growth** — без `tf.config.experimental.set_memory_growth(gpu, True)` TensorFlow занимает всю видеопамять; при наличии других GPU-процессов возможны OOM.
- **Медленная валидация** — walk-forward обучает 54 модели (18 на горизонт × 3); с Optuna прогон занимает значительное время на CPU.

### Код

- **Мета-learner не оптимизируется через Optuna** — гиперпараметры захардкожены в `config.py`; базовые модели оптимизируются, мета-слой нет.
- **Дата начала данных захардкожена** — `start_date = '2014-01-01'` не вынесена в `.env` или CLI; для тикеров с коротким листингом грузится пустой период в начале.
- **Логирование сырого ответа API** — `logger.info(f"RAW API RESPONSE: {resp.text[:500]}...")` пишет первые 500 символов; ответ может содержать чувствительные данные.

### Что можно убрать

| Что | Где | Причина |
| --- | --- | --- |
| `TINVEST_USE_GRPC` | `.env` | Не используется в v14 |
| `FINANCEMARKER_TOKEN` | `.env` | Источник данных не подключён |
| Логирование `RAW API RESPONSE` | `stock_modelv14.py` | Шум + потенциальная утечка данных |

### Что можно добавить

- **GPU memory growth** — добавить в инициализацию: `for gpu in tf.config.list_physical_devices('GPU'): tf.config.experimental.set_memory_growth(gpu, True)`
- **Configurable start date** — `DATA_START_DATE` в `.env` и `--start-date` в CLI
- **Оптимизация мета-learner'а** — включить в Optuna или хотя бы вынести параметры в `config.py`
- **Настраиваемый горизонт прогноза** — через `.env` или CLI (сейчас жёстко `[1, 2, 3]`)
- **Прогресс-бар для Optuna** — `show_progress_bar=True` или tqdm

### Рефакторинг (кандидаты на вынос из stock_modelv14.py)

| Файл | Функции | Причина |
| --- | --- | --- |
| `optuna_tuning.py` | `optimize_lstm_params`, `optimize_xgboost_params` | Полностью изолированы от пайплайна; Optuna-код меняется независимо |
| `data_loader.py` | `load_stock_data_moex_test` | Единственное место для замены источника данных (MOEX → другой) |

Не трогать пока:

- `run_backtest` — плотно завязан на локальные переменные и logger
- `prepare_and_train_model` — громоздкая сигнатура при выносе, мало выгоды

### Открытые вопросы

1. **GUI / параметры Optuna** — если для тикера нет сохранённых параметров, по умолчанию запускать оптимизацию.
2. **GPU использование** — при запуске не отображается информация о GPU и неясно, используется ли он.
3. **Способ демонстрации** — нативный Windows-интерфейс или небольшой веб-сайт для защиты.
4. **Масштаб сохранения графиков** — отдельные срезы за 2 недели и месяц; дополнительный график с ошибкой в RUB и %.
5. **Завершение программы** — завершать работу, не ожидая закрытия matplotlib-окон.
6. Сравнить с [похожим проектом](https://github.com/GorelikovMatvey/Stock-Forecast-ARIMAX).
